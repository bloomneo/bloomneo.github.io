Bloom CLI

Scaffold fullstack web, desktop (Electron), and mobile (Capacitor) apps with Feature-Based Component Architecture.
Quick start
npm install -g @bloomneo/bloom

bloom create my-app                    # web app
bloom create my-app userapp            # with auth + user management
bloom create my-app desktop-basicapp   # Electron desktop
bloom create my-app mobile-basicapp    # iOS + Android via Capacitor
Architecture

Feature-Based Component Architecture

Every app follows the same convention. Features live in features/ and are auto-discovered — no routing config, no manual registration. One pattern that works across web, desktop, and mobile.

src/
├── api/features/welcome/     # Backend feature — auto-registered
│   ├── welcome.route.ts      # Mounts at /api/welcome
│   └── welcome.service.ts
└── web/features/main/        # Frontend feature — auto-routed
    └── pages/
        └── index.tsx         # Renders at /
