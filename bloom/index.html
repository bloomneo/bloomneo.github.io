@bloomneo/bloom

One CLI.
Web, desktop, mobile.

Bloom scaffolds fullstack apps with Feature-Based Component Architecture. AppKit backend + UIKit frontend — wired up, configured, ready to run.

npm ↗ GitHub ↗
Install and create
npm install -g @bloomneo/bloom

bloom create my-app                      # web app (default)
bloom create my-app userapp              # auth + user management + admin
bloom create my-app desktop-basicapp     # Electron desktop app
bloom create my-app desktop-userapp      # Electron + auth + SQLite
bloom create my-app mobile-basicapp      # iOS + Android via Capacitor
After create
cd my-app
npm install     # also copies appkit/uikit docs into docs/
npm run dev     # starts frontend + backend concurrently

Runs on: http://localhost:5173 (frontend) and http://localhost:3000 (backend API).

Feature-Based Component Architecture

Convention over configuration

Drop a folder in features/ and it's automatically discovered — no config, no registration.

# Backend: drop a folder → auto-registered route
src/api/features/
└── invoice/
    ├── invoice.route.ts     # Auto-mounts at /api/invoice
    └── invoice.service.ts

# Frontend: drop a folder → auto-routed page
src/web/features/
└── invoice/
    └── pages/
        ├── index.tsx        # → /invoice
        └── [id].tsx         # → /invoice/:id
Project structure — basicapp
my-app/
├── src/
│   ├── api/                  # Express backend (AppKit)
│   │   ├── features/
│   │   │   └── welcome/
│   │   │       ├── welcome.route.ts
│   │   │       └── welcome.service.ts
│   │   ├── lib/api-router.ts
│   │   └── server.ts
│   └── web/                  # React frontend (UIKit)
│       ├── features/
│       │   └── main/pages/index.tsx
│       ├── shared/components/
│       └── main.tsx
├── docs/                     # Auto-generated after npm install
│   ├── appkit.md             # ← copied from node_modules/@bloomneo/appkit/llms.txt
│   ├── appkit-agents.md      # ← copied from node_modules/@bloomneo/appkit/AGENTS.md
│   └── uikit.md              # ← copied from node_modules/@bloomneo/uikit/llms.txt
├── AGENTS.md                 # AI agent conventions for this project
└── package.json
Mobile — run on device
cd my-mobile-app
npm install

# iOS simulator
npm run mobile:run:ios

# Android emulator
npm run mobile:run:android

# Build APK
npm run android:build
Desktop — run and package
cd my-desktop-app
npm install
npm run dev            # Electron window + Vite + Express

npm run build          # Build for production
npm run preview        # Preview production build in Electron
AI-ready by default

Every scaffolded project includes an AGENTS.md at the root documenting conventions, feature structure, and env vars — so AI coding agents start with full project context. Running npm install automatically copies the latest API references from installed packages into docs/.
