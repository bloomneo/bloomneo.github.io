Our principles
🧩

Simplicity

Clarity over complexity. Tools that are easy to learn, easy to use, and easy to leave if you outgrow them. Setup should take minutes, not days.

🔌

Autonomy

Loosely coupled modules. Use AppKit without UIKit. Use UIKit without Bloom. Take what you need — nothing forces you to take everything.

🤖

AI Innovation

Progressive Scaling

AppKit — one rule that matters most

Every module is xxxClass.get()

No constructors. No factories. No exceptions. Predictable, non-ambiguous, AI-agent friendly.

import { authClass, databaseClass, errorClass, loggerClass } from '@bloomneo/appkit';

const auth     = authClass.get();
const database = await databaseClass.get();
const error    = errorClass.get();
const logger   = loggerClass.get('api');

app.post(
  '/api/users',
  auth.requireLoginToken(),                 // 1. authenticate
  auth.requireUserRoles(['admin.tenant']),  // 2. authorize
  error.asyncRoute(async (req, res) => {
    if (!req.body?.email) throw error.badRequest('Email required');
    const user = await database.user.create({ data: req.body });
    logger.info('User created', { userId: user.id });
    res.json({ user });
  })
);
app.use(error.handleErrors()); // always last

  • authClassJWT tokens · role.level hierarchy (9 levels) · Express middleware
  • databaseClassPrisma client · multi-tenant filtering · per-org databases
  • securityClassCSRF · rate limiting · AES-256-GCM encryption · input sanitization
  • errorClassSemantic HTTP errors (400–500) · async route wrapper · centralized middleware
  • cacheClassMemory → Redis · getOrSet() · namespace isolation
  • storageClassLocal → S3/R2 · presigned URLs · file listing
  • queueClassMemory → Redis jobs · add(), process(), schedule(delayMs)
  • emailClassConsole → SMTP → Resend · sendTemplate() · batch send
  • eventClassMemory → Redis pub/sub · wildcard patterns · namespace isolation
  • loggerClassStructured logging · console / file / HTTP transports
  • configClassType-safe env access · getRequired() · validateRequired()
  • utilClass12 zero-dep helpers: slugify, uuid, chunk, debounce, clamp, truncate…

Bloom — convention over configuration

Drop a folder. Get a route.

No routing config. No manual registration. Same file-based auto-discovery on frontend and backend.

# Backend: folder → auto-registered Express route
src/api/features/invoice/
├── invoice.route.ts     # mounts at /api/invoice
└── invoice.service.ts

# Frontend: folder → auto-routed React page
src/web/features/invoice/pages/
├── index.tsx            # → /invoice
└── [id].tsx             # → /invoice/:id

Built for AI agents

Every package ships machine-readable docs.

LLMs hallucinate APIs for undocumented packages. Every Bloomneo package ships llms.txt and AGENTS.md inside the npm tarball. After npm install, the Bloom CLI copies the latest docs from node_modules/ into your project's docs/ folder — always in sync with the installed version.

llms.txt

Complete machine-readable API reference. Every method, signature, and example. Regenerated on every build from source — never drifts.

AGENTS.md

Rules for AI coding agents: always-do, never-do, canonical patterns. Start here to avoid hallucinated method names.

Auto-synced docs/

The postinstall script copies the latest API docs from installed packages into your docs/ folder on every npm install.
